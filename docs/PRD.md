创作者素材管理平台 — PRD v1.0
1. 业务背景
创作者在平台上传视频/素材文件时，单文件可达数百 MB。现有上传方式为普通 HTTP 上传，存在以下问题：

大文件上传易超时中断，失败后需重新开始
无法断点续传，用户体验差
重复文件（秒传）每次都需要完整上传，浪费带宽和时间
2. 目标
支持大文件分片上传、断点续传、秒传，上传成功率 > 99%。

3. 需求列表
优先级	功能	说明
P0	分片上传	文件切片后并发上传每个分片，服务端合并
P0	断点续传	客户端查询已上传分片，只传缺失部分
P0	秒传	服务端检测文件内容是否已存在，存在则直接完成
P1	文件管理	查询文件列表、删除文件
P1	文件下载	下载已上传的文件
P2	用户鉴权	JWT 登录认证，文件归属用户
4. 核心流程
4.1 上传流程
客户端 → 1. 发起初始化 (文件名+大小+哈希) → 服务端
       ← 2. 返回 upload_id / 是否秒传 / 分片信息
       → 3. 查询已上传分片 (断点续传)
       ← 4. 返回已接收分片列表
       → 5. 并发上传缺失分片
       → 6. 通知合并完成
       ← 7. 返回文件元信息
4.2 秒传逻辑
客户端计算文件的 SHA256 哈希
服务端查询哈希索引，存在则直接关联到用户，返回秒传成功
5. 非功能性需求
支持最大 1GB 单文件上传
分片大小 4MB
并发上传 Worker 数：客户端 4 个，服务端限制 64 个并发写入
上传完成后 5 分钟内清理临时数据
